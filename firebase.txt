Here is the complete guide to uploading your memorial site to GitHub Pages and connecting your custom domain.

### Step 1: Push the Code to GitHub
First, we need to create a repository on GitHub and upload the files.

1. Log in to [GitHub](https://github.com/) and click the **"+"** icon in the top right corner, then select **New repository**.
2. Name the repository (e.g., `fr-winson-memorial`).
3. Set the repository to **Public** (GitHub Pages requires public repos for free accounts).
4. Do **not** check the boxes for README, .gitignore, or license. Just click **Create repository**.

Since your files are already on your computer, you can run these commands in your terminal (inside the `memorial-fr-winson` folder). *If you'd like, I can run these local commands for you! Just let me know.*

```bash
# Initialize the git repository
git init

# Add all the files
git add .

# Commit the files
git commit -m "Initial commit for Fr. Winson Memorial"

# Change the main branch name to 'main'
git branch -M main

# Link it to your new GitHub repository (REPLACE with your actual GitHub URL)
git remote add origin https://github.com/YOUR_USERNAME/fr-winson-memorial.git

# Push the code to GitHub
git push -u origin main
```

### Step 2: Enable GitHub Pages
Now that your code is on GitHub, we turn on the hosting.

1. On your GitHub repository page, click the **Settings** tab (the gear icon near the top right).
2. On the left sidebar, scroll down and click **Pages**.
3. Under the **Build and deployment** section:
   - For **Source**, select **Deploy from a branch**.
   - Under **Branch**, select `main` (or `master`) from the dropdown, and leave the folder as `/ (root)`.
   - Click **Save**.
4. GitHub will now build your site. In about 1–2 minutes, it will display a message at the top saying: *"Your site is live at https://yourusername.github.io/fr-winson-memorial/"*. 
   *(Your site is now publicly on the internet!)*

### Step 3: Configure Your Custom Domain (e.g., www.frwinson.com)
To use a custom domain that you purchased (from GoDaddy, Namecheap, Google Domains, etc.), follow these steps:

**Part A: Tell GitHub about your domain**
1. Go back to the **Pages** section in your GitHub repository Settings.
2. Scroll down to the **Custom domain** section.
3. Type your custom domain (e.g., `frwinson.com` or `www.frwinson.com`) into the box and click **Save**.
4. GitHub will automatically create a file called `CNAME` in your repository. It will say "DNS check in progress" (this is normal).
5. Check the box that says **Enforce HTTPS** (it might take an hour or two before you can check this box—it gives your site the secure padlock icon).

**Part B: Tell your Domain Registrar about GitHub**
Log in to the company where you bought your domain (GoDaddy, Namecheap, etc.), go to your **DNS Settings / Manage DNS**, and add these records:

1. **Add A Records (Points the main domain to GitHub's servers):**
   Create four separate **"A" Records** with the Host set to `@` (or leave blank) and point them to these four GitHub IP addresses:
   - `185.199.108.153`
   - `185.199.109.153`
   - `185.199.110.153`
   - `185.199.111.153`

2. **Add a CNAME Record (Points the 'www' version to your GitHub site):**
   Create a **"CNAME" Record** with the Host set to `www` and point it to your GitHub pages URL: `YOUR_USERNAME.github.io`.

**That's it!**
DNS changes can sometimes take anywhere from 15 minutes to 24 hours to update globally. Once it updates, anyone typing your custom domain will see the beautiful memorial site.
